项目一：基础实战 —— 轻量级服务器性能监控看板

项目定位：核心技能串联与可视化呈现

关键词：Python自动化 · Linux系统 · 数据可视化 · 告警意识

· 项目简介：这是一个本地化部署的服务器监控工具。通过Python脚本自动化采集主机的核心性能指标（CPU、内存、磁盘、网络），并将数据持久化存储。项目最终提供了一个简洁的Web图表界面，用于实时查看资源使用历史和趋势。
· 核心职责/实现：
  1. 使用 psutil 库编写采集脚本，并通过 crontab 设置定时任务。
  2. 设计MySQL数据库表结构，使用 pymysql 实现数据的稳定写入与查询。
  3. 利用 Flask 框架快速搭建Web后端，并结合 ECharts 前端库渲染动态图表。
  4. 实现简单的阈值检查逻辑，当CPU或内存使用率持续过高时，在界面触发警告标识。
· 技术栈：Python Flask MySQL Linux Shell ECharts/Matplotlib
· 学习价值：打通“数据采集 → 存储 → 展示”的完整流程，理解运维数据化基础，培养对系统性能的洞察力和初步的告警意识。
· 展示建议：在GitHub的README中提供系统架构图、关键代码片段（如数据采集和入库逻辑）以及最终的Web界面截图。

项目二：进阶实战 —— LNMP微服务自动化部署与高可用架构

项目定位：现代运维工程化实践

关键词：Docker容器化 · Ansible自动化 · 服务编排 · 高可用

· 项目简介：本项目旨在解决传统应用部署繁琐、环境不一致的痛点。将一个模拟的博客应用（LNMP架构）及其依赖的服务（Redis缓存）全部容器化，并编写自动化脚本，实现在全新服务器上的一键部署与高可用配置。
· 核心职责/实现：
  1. 为Nginx、PHP-FPM、MySQL、Redis及自研应用分别编写 Dockerfile，构建标准镜像。
  2. 使用 docker-compose.yml 定义多容器应用的启动顺序与网络配置。
  3. 编写 Ansible Playbook，自动化完成目标服务器的Docker环境初始化、镜像拉取、服务启动及Nginx负载均衡配置。
  4. 在公有云（如阿里云ECS）上创建两台虚拟机，成功执行Playbook，验证跨节点的服务部署与访问。
· 技术栈：Docker Docker Compose Ansible Nginx Linux 公有云(ECS)
· 学习价值：掌握基础设施即代码（IaC）的核心思想，深入理解容器编排与自动化运维，亲手搭建一个具备生产环境雏形的高可用微服务集群。
· 展示建议：提供 Ansible Playbook目录结构、Docker Compose文件 以及在公有云控制台成功运行服务的效果截图。可以录制一个简短的 GIF动图，展示从执行Playbook到最终访问服务的全过程。

项目三：专业融合实战 —— 物联网设备数据云边协同监控平台

项目定位：运维技术与专业领域结合

关键词：物联网数据流 · 云原生监控 · MQTT协议 · Grafana可视化

· 项目简介：本项目模拟了一个真实的物联网场景。边缘设备（由Python脚本模拟）通过MQTT协议上报传感器数据，云端服务接收并存储这些时序数据，最终通过专业的监控大盘进行集中可视化展示与告警。
· 核心职责/实现：
  1. 编写设备模拟器，定时通过 paho-mqtt 客户端向 Mosquitto (MQTT Broker) 发布模拟的温湿度数据。
  2. 部署 Telegraf 作为数据收集器，订阅MQTT主题并将数据写入 InfluxDB 时序数据库。
  3. 使用 Grafana 连接InfluxDB，配置丰富的仪表盘，实时展示设备数据、历史曲线和地理分布（若模拟数据包含位置）。
  4. 在Grafana中为异常数据（如温度过高）设置告警规则，并模拟触发。
· 技术栈：MQTT InfluxDB Grafana Docker Python 时序数据库
· 学习价值：深入理解物联网核心通信协议与数据特点，掌握业界主流的云原生监控技术栈，完成从设备到云端的数据链路设计与实现。
· 展示建议：提供系统数据流架构图、Grafana仪表盘全景截图以及关键告警规则的配置截图。可以附上一段简单的设备模拟器代码。
